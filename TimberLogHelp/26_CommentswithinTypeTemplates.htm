Comments within Type Templates

 

Comments within a Type Template file can help to document the Type. They are often useful with complex Logbook Types, and if the Type file is ever modified in the future, they may help to clarify details and decisions made when the Logbook Type was originally created.

 

There are two ways to add comments to a Logbook Type file: Standard comments and the Comments Block

 

Standard comments are any lines of text outside of a defined Block. They are never saved into the logbook.

 

The Comments Block provides users with a less transient method of adding comments to a Type File. If the Embed_TypeFile setting is set to True, these comments will be embedded into the Logbook, and can be exported by users along with the rest of the Logbook Type File at any time. This block allows multiple lines of text, enclosed in the Comments Block elements.

 

Both types of comments are demonstrated in this example:

 

This is an example of a standard comment. It is not placed within any of the defined Blocks.

 

>>>*Comments*<<<

This is a Comments Block comment.

 

Because it is placed within the Comments Block, it will be embedded into the Logbook if the Embed_TypeFile setting is set to True.

>>>*Comments*<<<

 

Multiple instances of the Comments Block are allowed in each logbook type file. Keep in mind that the placement of these blocks may change if the Type File is embedded into a Logbook and later exported. These blocks are optional.
